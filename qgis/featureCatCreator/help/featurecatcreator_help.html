Feature catalog creator plugin

Overview

The Feature Catalog Creator plugin lets you create XML metadata files with ISO19110 norm, for a given data set.

XML tab

The XML tab is the place where you will find the XML metadata file content. You can do the following actions :

  • Enter a XML template file URL or browse for a local file, and load it
  • See and edit the XML content
  • Refresh the XML content according to informations given in the other tabs
  • Save the XML file

Global tab

The global tab enables the user to enter informations about the Feature Catalog and the Feature type.
As for Feature catalog, the user can provide the following informations :

  • Name
  • Scope
  • Version number

The name should be relatively short, whereas the scope can be lengthy.
You can load name, scope and version number from the XML content in the first tab. If you have loaded a template XML file, clicking on the "load from XML" button enables you not to write those informations by hand.

As for Feature type, the user can provide the following informations :

  • Name
  • Definition

The name should be relatively short, whereas the definiton can be lengthy.
You can load name, scope and version number from the XML content in the first tab. If you have loaded a template XML file, clicking on the "load from XML" button enables you not to write those informations by hand.

Fields tab

The form contains the following elements :

Attribute table button
This button shows the attribute table of the current layer
Data source
This combobox selects the layer you want to extract informations from
Current field
This combobox contains the current available fields as defined by the user. By default it is greyed out, until you click on "load from layer" or "load from XML".
Load from layer button
Click this button to load fields informations from selected layer's data
Load from XML button
Click this button to load fields informations from XML content (to reload saved informations from an XML file for example)
Classification
If this checkbox is active, when loading fields informations from the layer, extract values from data. Classification parameters are set in the spinboxes just below.
Field part
This part of the form displays informations about the currently selected fields
Field name
Specify the name for the selected field. When loading from layer data, this is the attribute column name.
Field type
Specify the type for the selected field. You can specify your own type, or select one of the pre-defined types. When loading from layer data, this is the attribute column type as defined in QGIS.
Field definition
Enter a definition for the selected field. When loading from layer data, this is the attribute column's comment as defined in QGIS.
Field cardinality
Enter a cardinality for the selected field. You can specify your own cardinality, or select one of the pre-defined cardinalities.
Values part
This part of the form displays the values found in the currently selected fields. This part is automatically filled with values found in XML content if "load from XML" is clicked, or with values found in the current layer's data if "load from layer" button is clicked, with classification activated. You can also fill the values from the data values by clicking on the "analyze" button.
Values number
This spinbox lets you set the number of values to read from the data source to find values.
Value occurence
This spinbox lets you set the number of occurences of a value which should be found to add this value in the table.
NODATA
This button lets you enter NODATA value(s), which will be ignored when filling the values table, and set in the XML file
Analyze
This button starts the layer's data analysis to find values, and fills the values table.
Values table
This table contains values for the current field. You can add a new value, delete a value or clear the table with the buttons below. You can edit values (label, codes and definitions) by double-clicking on an item.

Note on classification : The classification done when analyzing data is done this way. For vector layers, read the nth first lines, and if a value is found with #occ occurences or more, add to in the values table. For raster layers, reading values in order makes no sense, hence nth values are read at random in the current viewport. If a value is found with #occ occurences or more, add it to the values table.
